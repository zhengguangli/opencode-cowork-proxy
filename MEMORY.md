# 全局记忆 — 代码风格准则

## 代码风格一句话总结

**"兼顾人类直觉与 AI 易读，坚持强类型与防御性编程，通过精准命名、模块化解耦和高覆盖测试，构建逻辑自洽的健壮系统。"**

## 核心原则

- **人类与 AI 双友好**：变量与方法命名直白且暴露意图，降低心智负担，让 AI 辅助工具极速理解上下文逻辑
- **强类型与防御性**：错误扼杀在编译期或运行初期（Fail-Fast）
- **解耦与测试驱动**：高度可测试性和复用性
- **精准命名 + 模块化**：每个函数职责单一，依赖清晰

## 技术栈核心追求

- 后端：复杂业务逻辑，强类型保障
- 纯函数优先：无副作用，易测试
- 防御性编程：空值检查、边界处理、类型收窄
- 高覆盖测试：单元测试 + 集成测试

## 在本项目中的应用

- `src/translate/` 所有翻译函数为纯函数（无 fetch、无副作用）
- `src/auth.ts` / `src/cache.ts`：纯函数可独立测试
- 类型显式声明，避免 `any` 隐式逃逸
- 流翻译测试用 mock ReadableStream 验证事件序列
