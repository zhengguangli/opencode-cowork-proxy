主要设计决策和架构信息记录在以下位置：

- `docs/ARCHITECTURE.md` — 分层架构、路由机制、ADR
- `docs/DESIGN.md` — 设计哲学、字段映射、错误策略
- `docs/QUALITY_SCORE.md` — 品味规则、代码约束

### 何时创建设计文档

当满足以下条件时，应在 `docs/design-docs/` 下创建设计文档：

1. 引入新的大规模特性（ > 2 天工作量）
2. 改变核心翻译逻辑（新的格式对、新的上游协议）
3. 引入架构性变更（新的部署目标、新的路由模式）
4. 需要团队评审的决策
