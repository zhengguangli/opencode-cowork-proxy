本项目是一个技术工具（API 翻译网关），主要需求反映在以下文档中：

- `docs/PRODUCT_SENSE.md` — 用户场景、价值定位
- `docs/PLANS.md` — 路线图、待办
- `docs/FIXES.md` — 已知修复和回归

### 何时创建产品规格

当满足以下条件时，应在 `docs/product-specs/` 下创建产品规格文档：

1. 引入面向用户的新 API 端点或协议支持
2. 变更认证/授权模型
3. 引入配置参数或环境变量变更
4. 影响现有客户端兼容性的变更
