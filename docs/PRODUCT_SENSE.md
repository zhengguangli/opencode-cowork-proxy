# PRODUCT_SENSE — 产品感知

> **何时读此文件：** 理解项目的产品定位、核心价值和目标用户。新增贡献者或 reviewer 的必读文档。

## 定位

**OpenCode Cowork Proxy** 是一个 AI API 翻译网关，为 AI 工具链提供统一的中间代理层。它让不同协议的客户端可以无缝连接到不同的 AI API 提供商，无需客户端侧修改。

## 核心价值

1. **协议兼容性** — Claude Desktop 和 Claude Code 原生只支持 Anthropic Messages 协议。本网关将其翻译为 OpenAI Chat Completions 和 OpenAI Responses 协议，使 Anthropic 客户端可以连接到非 Anthropic 上游。反之亦然。

2. **灵活路由** — 通过 URL 前缀（`/go`、`/zen`）和 `X-Upstream-Url` 头，用户可以在多个上游之间切换。路径中嵌入模型名即可覆盖请求体中的模型字段。

3. **错误透明** — 上游的任何错误都原样透传，不受翻译逻辑干扰。客户端看到的错误就是上游返回的错误，无二次封装损失。

4. **零成本格式适配** — 流式、图片、工具调用、系统提示等高级特性在各个协议间完整映射。用户无需关心底层协议差异。

5. **多部署目标** — Cloudflare Workers edge network、Vercel serverless、macOS 独立二进制。按需选择部署方式。

## 用户画像

- **AI 开发者**：使用 Anthropic 客户端但需要接入非 Anthropic 上游
- **平台集成商**：需要统一代理层来管理多个上游
- **Claude Code 用户**：通过代理使用更经济的上游模型

## 竞品对比

| 特性 | OpenCode Proxy | 自建 nginx 转发 | 客户端 SDK 适配 |
|------|----------------|----------------|----------------|
| 协议翻译 | 内置 3 方向 9 模块 | 需自写翻译逻辑 | 无 |
| 格式感知路由 | URL 前缀 + 请求头 | 无 | 无 |
| 流式翻译 | 内置 SSE 翻译 | 需自实现 | 无 |
| 错误透明 | 原样透传 | 需配置 | 取决于 SDK |
| 视觉模型自动选择 | 上游感知 + 模型集合检查 | 无 | 无 |

## 非目标

- 不存储用户数据（纯转发，无持久化）
- 不进行请求改写（格式翻译外尽可能保持原样）
- 不提供前端 UI（API-only gateway）
