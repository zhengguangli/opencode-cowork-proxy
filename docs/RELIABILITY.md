# RELIABILITY

## 错误处理策略

### 上游错误分类

| 类别 | 处理 | 重试 |
|------|------|------|
| 4xx 客户端错误 | 透传上游响应到客户端 | 不重试 |
| 5xx 服务端错误 | 指数退避重试后透传 | 最多 2 次 |
| 网络错误 | 指数退避重试 | 最多 2 次（流请求除外） |
| 超时（AbortError） | 返回 499 响应 | 不重试 |

### 重试机制

```
MAX_RETRIES = 2
Base delay = 500ms
Delay formula = 500ms × 2^attempt + random_jitter(0-200ms)
Max delay = 10,000ms
```

**关键异常：** 流（SSE）请求不重试——SSE 流不可重放。

### 超时配置

| 超时类型 | 值 | 说明 |
|----------|-----|------|
| 非流请求 | 60s | 普通 API 调用 |
| 流连接 | 120s | 客户端断开或上游停流 |
| 模型列表获取 | 10s | 快照式查询 |

## 背压控制

流翻译器使用 `applyBackpressure()` 防止消费者滞后时无限缓冲：

```
当 controller.desiredSize ≤ 0:
  waitMs = min(|desiredSize| × 0.5, 100ms)
  等待 waitMs 后继续
```

适应 Cloudflare Workers 的受限内存环境。

## 幂等性与重复防护

- 代理本身不提供幂等性保证——依赖上游的幂等性
- 重试仅在 GET 和 non-streaming POST 上执行
- 多次重试可能产生重复请求，由上游去重

## 降级行为

| 场景 | 降级 | 用户可见影响 |
|------|------|-------------|
| 缓存不可用 | 跳过缓存，直接请求上游 | 模型列表响应慢 ~10s |
| Gzip 压缩失败 | 返回未压缩的 JSON | 响应稍大，影响可忽略 |
| body size 检查失败 | clone() 失败 | 跳过检查，放行请求 |
| 认证检查失败 | 返回 401 | 正常 |
